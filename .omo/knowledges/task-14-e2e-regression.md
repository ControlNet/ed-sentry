# Task 14 E2E Regression Test Design

- Sanitized replay coverage lives in `tests/replay.rs` and uses `assert_cmd::Command::cargo_bin("ed-afk-monitor")` against `tests/fixtures/journal_combat_bounty.log`; required fragments are upstream-style `Scan`, `Kill`, `Total Stats`, and `Monitor stopped`, with an explicit guard against the ANSI clear-current-line sequence `\u{1b}[2K`.
- Live regression coverage stays deterministic in `tests/live_tail.rs`: a temp `Journal.*.log` file is preloaded to EOF, partial appended JSON is buffered until newline, then complete appended events flow through `parse_journal_line`, `EventMonitor`, `NotificationDispatcher`, and `FakeNotifier` without sleeping or touching real Journal files.
- The ignored real Journal regression in `tests/real_journal_replay.rs` never copies or prints raw Journal lines. It opens `/home/ubuntu/Elite Dangerous` read-only, prioritizes known sample filenames, falls back to local `Journal.*.log`, applies parsed events to `SessionState`, and reports only aggregate counts for Bounty, ShipTargeted, MissionRedirected, ShieldState, FighterDestroyed, and HullDamage.
- If the real Journal folder is absent, unreadable, empty, or missing the desired local categories, the ignored test prints a sanitized skip/status message and exits successfully so CI and `cargo test --all` remain independent of private local data.
