# GUI WebUI Task 10 Emoji Sanitizer

Date: 2026-06-20

The Recent event feed can receive backend runtime status summaries generated by Rust terminal status helpers, for example `💥3600.0/h ⌚0s 🎯 0/2`. CLI terminal output should keep that formatting, but the WebUI design system forbids visible emoji icons.

The narrow fix is frontend display sanitation in `ui/src/components/dashboard/dashboard-helpers.ts` at `lineSafeText`:
- map known status glyphs to words: `💥 -> Kills`, `📦 -> Scans`, `⌚`/`⏱ -> uptime`, `🎯 -> missions`;
- remove emoji sequence markers and remaining pictographic/status glyph ranges;
- preserve existing control-character cleanup and `Journal.*.log -> Selected Journal file` redaction.

Evidence pattern for future gates:
- include a browser/DOM assertion, not only source grep;
- `ui/scripts/axum-smoke.mjs` now emits `EVENT_FEED_TEXT` and asserts no emoji/status pictographs in the Recent event feed;
- `scripts/smoke-webui.sh` dashboard scenarios launch without `--no-status-line` so backend runtime status rows are present in WebUI smoke.
