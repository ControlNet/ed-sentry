# User-facing README direction

- The README should be short and user-facing: product purpose, feature list, GitHub Releases download link, which executable to run, quick start, minimal `config.toml`, common CLI commands.
- Avoid putting internal development checks, privacy scan commands, full release workflow mechanics, roadmap exclusions, or fixture policy in the main README.
- The current README presents GitHub Releases as the user download location: `https://github.com/ControlNet/ed-sentry/releases`.
